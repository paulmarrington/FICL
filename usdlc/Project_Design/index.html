Vision

Interactive Configuration

Provide the power of a computer language without the size and complexity needed to write full-blown programs. It only needs to have enough complexity to be able to make configuration changes and implement workflow.

Vision

Small footprint

Be able to run within applications on mobile devices, desktop systems and servers.
Vision

Ubiquitous

Be able to run in any framework that needs interactive configuration. Interpreters such as Python and Ruby already have these facilities within their own language structure.
Vision

Integration

Be able to interact with the underlying system - typically by being able to create object instances and call methods.
Vision

Workflow ready

Be able to make decisions based on available data.
Vision

Easy to install

Make installation into a new system as simple as possible - typically including one module without external dependencies and making one call on program start.
